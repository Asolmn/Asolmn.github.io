Mariadb主从复制的一次报错

2019-05-22 #Linux
slave体现不了master的数据更新 刚刚解决，趁机水一篇博客 报错体现 Slave_IO_Running:Connecting 查看一些报错信息 error connecting to master 'slave@192.168.116.22:3306' - retry-time: 60 retries: 86400 message: Host '192.168.116.33' is not allowed to connect to this MariaDB server 问题出在访问连接权限上面。 解决方法: grant replication slave on *.* to 'slave'@'192.168.116.%'; grant replication user_name on *.* to 'user_name'@'ip' user_name用户名，ip可以使用%通配符

Samba报错

2019-05-08 #Linux
记一次samba的报错，报错体现在远程挂载超时，始终不能访问，其问题在于域名解析。 Samba配置文件内容如下 [webdata] path = /data/web_dat writable = yes hosts allow = 192.168.101.0/24 hosts deny = all public = yes 使用用户为apache，添加用户 smbpasswd -a apache 重启服务 systemctl restart nmb smb 报错内容如下 使用命令smbclient -L //192.168.101.22/webdata -U apache用来测试 报错显示: protocol negotiation failed: NT_STATUS_IO_TIMEOUT 造成原因，是因为服务器本身是DNS服务，所以要对dns服务进行停止systemctl stop named 第二种的话，则是/etc/resolv.conf文件中的nameserver参数为114.114.114.114，将其注释掉就行 还可以在客户端的host文件中添加服务端的主机名记录 例如:添加192.168.116.22 serverA.rj.com
