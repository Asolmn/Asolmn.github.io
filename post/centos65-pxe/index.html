Centos6.5 PXE

2019-05-06

Centos6 PXE自动远程安装服务+客户端完全自动化脚本

环境:

Server:10.80.60.105

Clients:10.80.60.106

今天因文档混乱，需要总结一下，在此情况下这篇博客就怎么诞生了（本人文笔不好，博文生硬，请见谅(嘿嘿)）。

PXE是一个远程网络安装服务，通过dhcp的网络自动分配，vsftp提供安装的镜像文件，然后tftp服务负责boot部分。可以通过编写ks.cfg脚本文件实现客户端的完全自动化。当然PXE的镜像提供服务并没有一定要求vsftpd，HTTP和NFS一样可以胜任这项工作。但是注意vsftpd比起其它两个服务j具有不错的权限设置。
接着废话少说，直接开始上实操吧

Server----操作:

安装服务包

yum install vsftpd dhcp  tftp-server syslinux -y

开始进行dhcp的操作 输入命令:

cp /usr/share/doc/dhcp-4.1.1/dhcpd.conf.sample /etc/dhcp/dhcpd.conf

开始修改dhcp配置文件

vim /etc/dhcp/dhcp.conf

启动dhcp

service dhcpd restart

dhcp至此配置完毕 开始配置tftp服务内容

vim  /etc/xinetd.d/tftp

输入如下命令：

mkdir /tftpboot  创建tftp服务主目录

复制挂载在mnt目录下镜像的引导程序

cp -rp /mnt/isolinux/* /tftpboot

pxelinux.0是一个bootload 可以通过它来装载内核(load kernel)

cp -rp /usr/share/syslinux/pxelinux.0 /tftpboot

进入到tftpboot目录 在此目录下创建pxelinux.cfg目录，然后将isolinux.cfg文件复制到pxelinux.cfg中，并改名为default

cd /tftpboot
mkdir pxelinux.cfg
cp isolinux.cfg pxelinux.cfg/default

启动xinetd

service xinetd restart

到这一步tftp的已经基本完成，开始vsftpd的配置 修改vsftpd配置文件vsftpd.conf

vim /etc/vsftpd/vsftpd.conf

将系统镜像复制到vsftpd服务的匿名用户下(注意本实验使用的是vsftpd的匿名用户默认目录)

cp -rp /mnt/* /var/ftp/pub

开始添加ks.cfg自动化脚本 将root目录下的anaconds-ks.cfg当模板

cp /root/anaconda-ks.cfg /var/ftp/pub/ks.cfg
vim /var/ftp/pub/ks.cfg

vsftpd部分内容基本配置完毕 对目录权限进行修改，以防客户端安装过程中会出现问题

chmod 777 /var/ftp/pub/*

重启vsftpd服务

service vsftpd restart

server端的最后一步就是配置default文件了，这个是客户端引导的关键

vim /tftpboot/pxelinux.cfg/default

至此server的配置就全部完成 最后注意不要忘记关闭防火墙和selinux

service iptables stop
setenforce 0

Clients-----操作

创建一个测试的虚拟机 开机进入网络安装模式(开机按esc健) 然后就不需要动了，会自动到安装系统步骤 安装完成后就是一个全新的系统了

End
